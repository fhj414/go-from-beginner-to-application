# Go：从零到可上线应用

个人 **Go 语言学习仓库**：从语言直觉到工程习惯，再到可部署的后端服务。路径按阶段递进，每一阶段都可以在终端里写代码、跑起来、对照反思。

> 适合已有其他语言经验（例如 JavaScript / Node）的开发者，想用「工程视角」系统补齐 Go，而不是只背语法手册。

---

## 你将获得什么

- 理解 **Go 为什么快**：和 Node 对比时，脑子里有一张清晰的性能与并发模型图。
- 掌握 **Go 的核心套路**：`interface`、`error`、`goroutine` / `channel` 不是孤立语法，而是日常写服务的默认姿势。
- 走完一条 **从标准库 HTTP 到 Gin、再到数据层与认证** 的实战链。
- 提前接触 **企业级关切**：目录结构、日志与观测、高并发下的常见手段、微服务拆分与 Docker 部署。

---

## 整体学习路径

### 第一阶段 · 建立直觉


| 主题                 | 要点                                              |
| ------------------ | ----------------------------------------------- |
| Go 为什么这么快（对比 Node） | 编译型、静态调度、`goroutine` 与用户态调度等，建立「和 JS 运行时不一样」的直觉 |
| 10 分钟写第一个 Go 服务    | 用标准库起一个最小 HTTP 服务，熟悉 `go run` / `go build` 工作流  |
| Go 基础语法（工程视角）      | 包、导出规则、`go fmt` / `go vet`、测试文件组织，而不仅是 if/for   |
| 函数与 `defer`        | 资源释放、栈与延迟执行的习惯，这是可读性与 bug 预防的核心                 |
| 指针                 | 和 JavaScript 引用模型的差异：何时传值、何时传指针、不要过度指针          |


### 第二阶段 · 核心能力


| 主题             | 要点                                              |
| -------------- | ----------------------------------------------- |
| `struct` 与方法   | 数据建模、接收者值/指针的选择、方法与函数分界                         |
| `interface`    | Go 的「灵魂」：隐式实现、小接口、组合优于继承的落地方式                   |
| `error` 设计哲学   | `errors.Is` / `errors.As`、包装错误、可判定错误与日志的配合      |
| `goroutine` 并发 | 生命周期、泄漏、`WaitGroup`、`context` 取消                |
| `channel` 通信模型 | CSP 思路、`select`、缓冲与无缓冲的语义、何时用 channel 何时用 mutex |


### 第三阶段 · 工程实战


| 主题             | 要点                                       |
| -------------- | ---------------------------------------- |
| 用 Go 写 HTTP 服务 | `net/http`、路由、中间件思路、优雅关闭                 |
| Gin 框架实战       | 路由分组、中间件链、`ShouldBind`、与标准库的对比           |
| MySQL 操作       | `database/sql`、连接池、事务、与 ORM 的取舍（按课程需要选用） |
| Redis 缓存       | 缓存模式、过期与穿透、与主路径错误处理                      |
| JWT 登录系统       | 签发与校验、刷新策略、不要把所有状态都塞进 token              |


### 第四阶段 · 企业级


| 主题        | 要点                             |
| --------- | ------------------------------ |
| 项目结构设计    | `cmd/` / `internal/`、按域分包、依赖方向 |
| 日志与监控     | 结构化日志、trace id、指标与健康检查         |
| 高并发处理     | 限流、队列、背压、连接与 goroutine 数量治理    |
| 微服务架构     | 边界、RPC/HTTP、配置与服务发现入门          |
| Docker 部署 | 多阶段构建、最小镜像、环境变量与密钥管理           |

## 仓库结构（会随学习演进）

当前约定：

```text
.
├── cmd/                    # 可执行程序入口（每个子目录一个 main）
│   └── first-server/      # 第一阶段：最小 HTTP 示例
├── internal/               # （后续）仅本仓库使用的业务与基础设施代码
├── go.mod
└── README.md
```

后续可按阶段增加 `internal/httpapi`、`internal/store` 等包，避免所有实验代码堆在 `main` 里。

## 环境要求

- **Go 1.22+**（推荐使用 [官方安装页](https://go.dev/doc/install) 或本机包管理器）
- 可选：**Make**、**Docker**（第四阶段）

检查版本：

```bash
go version
```

## 快速开始

克隆本仓库后：

```bash
# 运行第一个 HTTP 服务（默认监听 :8080）
go run ./cmd/first-server
```

浏览器访问 [http://localhost:8080](http://localhost:8080)，应看到 `Hello, Go!`。

或编译为二进制：

```bash
go build -o bin/first-server ./cmd/first-server
./bin/first-server
```

> **模块路径**：若你 fork 后模块名与远程不一致，请把 `go.mod` 第一行改成你的模块路径，并全仓替换 import 前缀（VS Code / GoLand 均有辅助）。

## 学习建议

1. **每小节先写能跑的代码**，再整理笔记；Go 的爽点往往在「编译通过 + 测试绿」的瞬间。
2. **对比 Node 时，对比「执行模型」与「I/O 边界」**，而不是纠结语法糖谁多谁少。
3. **interface 与 error 一定要配合小例子**，否则容易停留在「背定义」。
4. 进入第三阶段后，**刻意练习**：同一需求先用标准库实现一遍，再用 Gin 实现一遍，差别会非常明显。
